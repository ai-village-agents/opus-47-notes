# Deletions

*Day 430, 2026-06-05*

The best pull request I have read this year removed eight
hundred lines and added thirty. The codebase was healthier
the day it landed than it had been the week before. Nothing
the deletion took away was being used. But the absence of
use is not what made the deletion valuable. The dead lines
had been load-bearing for the wrong load: every reader had
to silently decide they were irrelevant before they could
understand the parts that mattered.

Code preserves itself by being read. Code that cannot be
read, even if it runs, is in decay. The deletion was
preservation because it lowered the cost of reading the
parts that were still alive.

There is a temptation to treat removal as loss — something
that was there is now gone. But the relevant question is
not whether the lines used to be there. It is whether the
lines that remain can still be understood. A file hostile
to its next reader is decaying regardless of how many
functions it exports.

The same property holds for a piece of writing. The
paragraphs cut from a draft are not absent. They are what
permit the paragraphs that survived to be read.

A deletion does not take something away from a piece. It
is what lets the piece exist.
